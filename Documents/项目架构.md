# 项目架构

面向维护者、调试者和 Mod 开发者。本文档描述 HighPerformanceBlacksmith (HPB) 的架构，重点标注与 BlacksmithFramework (BF) 的差异。

---

## 1. 项目概览

多个 .NET 8/9 项目组成的方案。

```
BlacksmithCore  (自包含，无外部项目依赖)
  ↑
BlacksmithClient  BlacksmithServer  AIPVPPlatform

ModExamples  (→ BlacksmithCore，示例 Mod)
ClapSourceGenerators  (Roslyn 编译时分析器，独立项目)
```

BlacksmithClient 启动 ASP.NET Core 本地站点，用 `wwwroot` 托管静态页面，通过 `/api/*` 端点操作同一进程里的 `GameInstance`。

目录结构：

```
Project/
├── Clap/
│   └── ClapSourceGenerators/   # Roslyn 增量源生成器
├── Blacksmith/
│   ├── BlacksmithClient/       # Web 宿主（入口）
│   ├── BlacksmithCore/         # 游戏引擎
│   │   ├── Infra/
│   │   │   ├── Unit/           # ClapRoundClock, ClapSharedFlag, ClapStateVar
│   │   │   ├── Utils/          # DllLoader, ModLoader
│   │   │   ├── Models/         # Body, Community, Health, Skill, TurnContext, etc.
│   │   │   │   └── Components/

│   │   │   │       └── AnalyzedObjects/  # DefenseEntity, EffectEntity
```

## 2. 核心基础设施

BlacksmithCore 自包含所有基础设施，不再依赖外部 Clap 库。核心类型：

### 可扩展枚举：`BlacksmithEnum<T, TMemberAttribute>`

C# 原生 `enum` 编译后不可追加。`BlacksmithEnum` 通过"运行时创建成员 + 全局递增唯一 ID"解决。成员通过 `[IsBlacksmithEnumMember(priority)]` 特性的 `public` 无参实例方法自动注册。`Create(name, priority)` 直接覆盖同名成员——这是 Mod 修改枚举优先级的基础。启动完成后 `CloseFactory()` 关闭工厂。

### 技能包：`SkillPackageBase`

编译时由 Roslyn 增量源生成器扫描 `private` 实例方法并配对：

- `XxxCheck(ISkillContext)` → `bool`：可用性检查
- `Xxx(ISkillContext)` → `IDSLSourceFile`：技能实现

诊断级别为 **Error**：未配对的 Check 方法或错误的返回类型会阻止编译。

### 多包技能管理：`Skill`

管理多个技能包，提供统一入口：`AddPackage` / `AddSkill` / `RemoveSkill` / `TryDeclare` / `Declare`。`PackageContainer` 携带 `ClapSharedFlag` 实现包的启用/禁用。

### 实体组件：`Body`

按类型键存储组件的容器。`Get<T>()` 取出组件。`Update()` 遍历 `IUpdatePerRound` 组件。

`Body` 注入 7 个组件：`Skill`、`Health`、`Defense`、`Resource`、`Effect`、`TurnContext`、`Community`。

### 回合上下文：`TurnContext`

> **HPB 变更**：泛型参数从 `IResolution` 改为 `IAnalyzableData`，不再继承单独的泛型基类。

按类型缓存 `List<TAnalyzableData>`，同时实现 `IUpdatePerRound` 以管理预处理钩子的生命周期。核心方法：

- `WriteAnalyzableData(analyzableData)`：先运行预处理钩子，再写入列表
- `Get<TAnalyzableData>()`

注册四种数据：`AttackAnalyzableData`、`DefenseAnalyzableData`、`ResourceAnalyzableData`、`EffectAnalyzableData`。每个数据携带一个 `ClapRoundClock`，执行时只处理 `Clock.IsRinging` 的条目，否则调用 `Clock.RoundPass()` 推进。

### 判定管线

- **`IDSLSourceFile`**：定义 `Compile(JudgeRuleManager?)` → `Intent` 契约，含 `IsPassive` 和 `Move()` 方法
- **`Judger`**：判定主循环，被动技能放在非被动技能之后编译
- **`JudgeRuleManager`**：9 阶段判定规则管理
- **`Intent`**：包含 `Action<Community> Execute` 的执行单元

### 工具

- **`DllLoader`**：扫描目录下所有 `*.dll`，提供 `LoadByType<T>()` 和 `LoadStaticByAttribute()`
- **`BlacksmithEnumRegistry`**：枚举注册表
- **`ClapSourceGenerators`**：Roslyn 增量源生成器，含 `AnalyzersRegistrationGenerator` 用于自动注册 `[IsAnalyzer]` 方法
- **`ClapRoundClock`**：统一的多回合生命周期时钟
- **`ClapSharedFlag`**：引用计数式启用/禁用开关
- **`ClapStateVar<T>`**：跨回合状态包装器

## 3. 启动流程

入口：`Program.cs`。

```
ModLoader.Initialize(BaseDirectory)
  1. 读取 .blacksmith/mod.json 解析 Mod 目录
  2. DllLoader 扫描目录下 *.dll
  3. 加载 BlacksmithEnum 子类和 IsBlacksmithEnumModifier 静态类 → 注册枚举 → CloseFactory()
  4. 扫描 SkillPackageBase 子类 → 注册主职业名 → 注册修改器
  5. 扫描 [IsAnalyzer] 静态方法 → 注册到 AnalyzerRegistry（新增步骤）
  6. 扫描 ISkillMetadata 实现类 → 收集元数据
  7. AddModOnInit 将修改器技能合并到目标包 + 调用 modifier.Bind(package)
→ LocalHost.Start(strategies)
  配置 ASP.NET Core，绑定 http://localhost:5000，映射 API，打开浏览器
```

## 4. 运行时核心对象

### GameInstance

一局游戏的根容器，实现 `ISkillContext` 和 `ISudoOperations`。核心属性：

- `Community Player` / `Community Enemy`：双方阵营
- `Judger Judger`：回合裁判
- `GameHistory History`：历史记录
- `GameMetadata Metadata`：全局元数据

### Community

`Body Focus` + `List<Body> SummonList`。支持召唤/分身等多角色机制：

- `AddTransform(action, delayRounds, remainingRounds, isInfinite)`：注册定时行为
- `AddCallbackKilled(body, callback)`：注册召唤物死亡回调
- `Update()`：清理已死亡召唤物 → 推进 Transform 时钟 → 更新 Focus 和各 Summon Body

### Body → BodyView

`Body` 通过 `Get<T>()` 访问 7 个组件。`GetView()` 返回 `BodyView` 供前端使用。

### 组件一览

| 组件 | 职责 | HPB 变更 |
|---|---|---|
| `Health` | HP / MHP，`GainHP` / `LoseHP` / `GainMHP` / `LoseMHP` | 无 |
| `Resource` | 资源系统，`Check` / `Use` / `Gain` / `Query` | 无 |
| `Defense` | `List<DefenseEntity>`，`Add`（同类型且 `CanMerge` 则 `Merge()`），`Update` 清理过期防御 | `DefenseBase` → `DefenseEntity` |
| `Effect` | `List<EffectEntity>`（每个持有 `ClapRoundClock`），`Update` 清理 `Clock.IsDead` 的过期效果 | `EffectEntity` 实现 `IAnalyzableData` |
| `TurnContext` | 四种 AnalysableData 缓存，通过 `ClapRoundClock` 延迟处理，预处理钩子（带独立 Clock 生命周期），`Update()` 清理过期预处理 | 实现 `IComponent<Body>` |
| `Skill` | 多包技能管理 | 无 |
| `Community` | Focus + SummonList | 无 |

## 5. 技能系统

### SkillPackageBase

编译时源生成器自动配对 `XxxCheck` / `Xxx`。技能名转为小写。

### MainProfession 与 ProfessionModifier

**ModifierBindingGenerator**：Roslyn 增量源生成器，使 Modifier 能够直接读写目标职业的私有状态变量，零反射开销。

### SkillMetadata 技能元数据

HPB 扩展了元数据属性：

| 属性 | 说明 |
|---|---|

| `[HasDefense]` | 标记为防御技能 |
| `[HasResource]` | 标记为资源技能 |
| `[HasRecovery]` | 标记为恢复技能 |
| `[HasBuff]` | 标记为 Buff 技能 |

## 6. 技能 DSL

技能不直接修改战场状态，而是先翻译为 `IDSLSourceFile`（`SourceFile`）。

> **核心差异**：HPB 的 DSL 将 `Resolution` 改为 `AnalyzableData`，所有 `WriteXxx` 方法签名均与 BF 不同。详见 [架构升级说明](./架构升级说明.md#6-dsl-write-方法签名变化)。

### SourceFile 结构

- `_owner`：`Community`
- `_sentences`：`List<Sentence>`，每条包含 `Structure`（`Action<Community>`）、`SentenceType`、`StructureType`
- `_rhetoricCache`：`Stack<Sentence>`（用于 `WithRuntime` / `WithComplieTime`）
- `_mutationsOnCompile`：动态注册回调列表（`List<List<ICallbackOnJudge>>`）
- `IsPassive`：bool，标识此 SourceFile 是否来自被动技能

### Pen 模式

```csharp
using Pen = Func<DSLforSkillLogic.SourceFile, DSLforSkillLogic.SourceFile>;

Pen pen = sf => sf
    .UseResource(1, ResourceType.Instance.Iron())
    .WriteAttack(3, AttackType.Instance.Physical())
    .WithBloodSuck(0.66f);

return DSL.Create(sc.Self, pen);
```

### 主要语句

| 语句 | 签名 | 可移动 | 说明 |
|---|---|---|---|
| `WriteAttack` | `(int power, AttackType.CEValue, int delayRounds=0, float aPFactor=1f, string analyzerKey=...)` | ✓ | 写入 `AttackAnalyzableData` |
| `WriteDefense` | `(int power, DefenseEntity defense, int delayRounds=0, string analyzerKey=...)` | ✓ | 写入 `DefenseAnalyzableData` |
| `WriteResource` | `(float power, ResourceType.CEValue type, int delayRounds=0, string analyzerKey=...)` | ✓ | 写入 `ResourceAnalyzableData` |
| `WriteEffect` | `(EffectType.CEValue, EffectTargetType.CEValue, ClapRoundClock entityClock, string analyzerKey, int delayRounds=0, float power=0)` | ✓ | 写入 `EffectAnalyzableData` |
| `WriteRecovery` | `(int power)` | ✓ | 直接回复 HP |
| `WriteCompileTime` | `(Action<Community> action)` | ✓ | 编译时执行（**替代 BF 的 WriteFree**） |
| `UseResource` | `(float need, ResourceType.CEValue type, bool ifCommonOnly=false)` | ✗ | 消耗资源 |
| `LoseHP` | `(int loss)` | ✗ | 扣血 |
| `LoseMHP` | `(int loss)` | ✗ | 扣最大 HP |
| `AddMark` | `(EffectEntity entity)` | ✗ | 添加标记效果 |

### 攻击修辞（仅 `AttackFile`）

- `WithBloodSuck(percent)`：攻击结算后按 `TotalDamage * percent` 吸血
- `WithInterupt()`：首次命中护甲时移除 Iron/Gold_Iron/Magic 资源数据
- `WithRuntime(stage, analyzerKey)`：在指定攻击阶段执行分析器（**替代 BF 的 `WithFree(stage, action)`**）
- `WithComplieTime(Action<AttackAnalyzableData>)`：编译时修改攻击数据

### Compile(judgeRuleManager)

> **HPB 变更**：参数从 `Judger` 变为 `JudgeRuleManager?`。

展开修辞缓存 → 注册动态 `ICallbackOnJudge` → 合并所有句子为单个 `Action<Community>` → 返回 `Intent`。

### Move() 所有权转移

`SourceFile.Move(newOwner, filter)`：更换 `_owner`，剥离 filter 中指定 `SentenceType` 的句子。用于 Association 模式。

## 7. Analyzer 体系（HPB 新增）

这是 HPB 相对于 BF 的最核心新增基础设施。

### 架构概览

```
┌─────────────────────────────────────────────────┐
│                  AnalyzerRegistry                │
├─────────────────────────────────────────────────┤
│  .DSL          AnalyzerRegistry<Action<C,C,     │
│                IAnalyzableData>>                 │
│  .Defense      AnalyzerRegistry<Action<C,C,     │
│                DefenseEntity, AttackAnalyzable>> │
│  .JudgeCallback AnalyzerRegistry<Action<C,C>>   │
│  .Universal    UniversalRegistry                │
└─────────────────────────────────────────────────┘
          ▲                ▲
          │ 注册            │ 查找
          │                │
  ┌───────┴──────┐  ┌──────┴──────┐
  │ [IsAnalyzer] │  │ AnalyzerKey │
  │ 静态方法      │  │ 字符串       │
  └──────────────┘  └─────────────┘
```

### AnalyzerType 枚举

```csharp
public enum AnalyzerType
{
    DSL,           // 通用：Attack/Defense/Resource/Effect 写入与执行
    Defense,       // 防御专属：签名含 DefenseEntity + AttackAnalyzableData
    JudgeCallback, // 判定回调
    Universal      // 泛型注册
}
```

### [IsAnalyzer] 属性

```csharp
[AttributeUsage(AttributeTargets.Method, AllowMultiple = false, Inherited = false)]
public class IsAnalyzer : Attribute
{
    public readonly AnalyzerType Type;
    public IsAnalyzer(AnalyzerType type = AnalyzerType.DSL) { Type = type; }
}
```

标记了 `[IsAnalyzer]` 的静态方法在 Mod 加载时被 `AnalyzersRegistrationGenerator`（源生成器）自动扫描并注册到对应的 `AnalyzerRegistry` 中。

### 标准分析器

**StandardAnalyzers.DSL.cs**:
| Analyzer | 功能 |
|---|---|
| `DefaultAttack` | 攻击流程：遍历防御 → AP 因子 → 扣血 |
| `DefaultBloodSuck` | 吸血（由 WithBloodSuck 挂载到 OnEnd） |
| `DefaultInterupt` | 打断（由 WithInterupt 挂载到 OnHitArmorFirstTime） |
| `DefaultResource` | 资源获取 |
| `DefaultDefense` | 防御添加 |

**StantardAnalyzers.Defense.cs**:
| Analyzer | 功能 |
|---|---|
| `DefaultArmor` | 护甲吸收伤害（护甲值同步减少） |
| `DefaultReduction` | 减伤（值不减少，纯吸收） |
| `ThornReduction` | 荆棘减伤（反弹一半伤害为魔法攻击） |
| `MagicalImmunity` | 魔法免疫 |
| `PhysicalImmunity` | 物理免疫 |
| `PercentageReduction` | 百分比减伤 |

### 关键收益

- **新增类型极简**：只需声明数据类 + 注册 Analyzer
- **行为可动态替换**：更换 `AnalyzerKey` 即可改变行为
- **计算逻辑集中审计**：所有逻辑在 `StandardAnalyzers` 中一目了然
- **性能优化**：纯数据更利于缓存和批量处理，减少委托分配和 GC 压力

## 8. 判定引擎

> 完整 9 阶段执行流程见 [判定流程](./判定流程.md)。

### 完整链路

```
SkillPackage → IDSLSourceFile → Compile(judgeRuleManager) → Intent
→ TurnContext (AnalyzableDatas) → JudgeRuleManager (9-Stage Chain) → Body.GetView()
```

### 9 个判定阶段（HPB）

> **HPB 变更**：从 BF 的 11 阶段精简为 9 阶段。移除 `OnEffectSwaping` 和 `OnAttackSwaping`。

| 阶段 | 优先级 | 默认行为 |
|---|---|---|
| `OnBegin` | 0 | 空 |
| `OnApplyingEffect` | 8 | 执行 EffectAnalyzableData（创建 EffectEntity） |
| `OnEffectTaking_AfterAnalyzableDataWritten` | 16 | 执行 AfterAnalyzableDataWritten 效果 |
| `OnAttackCanceling` | 32 | 按优先级 1:1 对消攻击 |
| `OnEffectTaking_AfterTransport` | 64 | 执行 AfterTransport 效果 |
| `OnApplyingOthers` | 128 | 依次执行：防御 → 攻击 → 资源 |
| `OnUpdating` | 256 | `Community.Update()` |
| `OnEffectTaking_AfterResult` | 1024 | 执行 AfterResult 效果 |
| `OnEnd` | 2048 | 空——留给动态注册 |

### 执行方式

> **HPB 变更**：执行从 `data.Execute(body)` 委托调用变为 `AnalyzerRegistry` 调度。

```csharp
// 旧 (BF)
resolution.Execute(body);

// 新 (HPB)
AnalyzerRegistry.DSL.Get(analyzableData.AnalyzerKey)(player, enemy, analyzableData);
```

### 动态注册核心类型

| 类型 | 职责 |
|---|---|
| `ICallbackOnJudge` | 接口，定义 `AnalyzerKey`（string）、`Stage`（`JudgeStage.CEValue`）、`Clock`（`ClapRoundClock`）、`IsPlayer`（bool） |
| `ModifierCallback` | 实现 `ICallbackOnJudge`，额外携带 `ModifierOrder`（`Before`/`After`），作为阶段核心规则的修饰器 |
| `OverrideCallback` | 实现 `ICallbackOnJudge`，直接替换阶段的默认规则 |

> **HPB 变更**：`ICallbackOnJudge` 不再包含 `JudgeRule` 委托字段，改为 `AnalyzerKey` + `IsPlayer`。执行时通过 `AnalyzerRegistry.JudgeCallback.Get(rule.AnalyzerKey)` 获取方法。

### StageRuleContainer 执行顺序

```
Before Modifiers（仅 IsRinging 的）
→ 核心规则（Override 或 Base，通过 AnalyzerRegistry.JudgeCallback 调度）
→ After Modifiers（仅 IsRinging 的）
→ Update()：移除 IsDead 规则，RoundPass() 推进
```

## 9. 防御系统（HPB）

> **HPB 变更**：完全重构。`DefenseBase` 被 `DefenseEntity` 取代。

所有防御继承 `DefenseEntity`：`AnalyzerKey`、`Type`（决定优先级）、`Power`、`Clock`、`CanMerge`、`Merge(DefenseEntity)`、`Update()`。**没有 `Work()` 方法**——计算逻辑在 `StandardAnalyzers` 中。

| 防御类型 | 优先级 | 特性 |
|---|---|---|
| `PhysicalImmunity` | -16 | 物理免疫 |
| `MagicalImmunity` | -16 | 魔法免疫 |
| `RealReduction` | 0 | 真实伤减 |
| `ThornReduction` | 8 | 刺盾，吸收物理攻击时反弹 50% 为魔法伤害 |
| `CommonReduction` | 16 | 普通伤减 |
| `PercentageReduction` | 可变 | 百分比减伤 |
| `CommonArmor` | 128 | 普通装甲，耐久型 |

### 攻击结算（DefaultAttack 分析器）

1. 真实伤害 → 跳过防御直接扣 HP
2. 否则按防御优先级遍历，APList 中的防御类型伤害乘以 `APFactor`，穿透后还原
3. 扣 HP 前触发 `OnHitBody` 阶段钩子，扣除后触发 `OnEnd`
4. 阶段钩子：`OnHitArmorFirstTime`（首次命中护甲）、`OnHitBody`（穿透所有防御后）、`OnEnd`（结算完毕）

## 12. 典型扩展路径

1. **新增职业**：继承 `MainProfession` → 写技能方法 → 转职方法标 `[IsProfessionSkill]`
2. **新增资源/攻击类型**：`[IsBlacksmithEnumModifier]` 静态类 → 扩展枚举
3. **新增防御**：扩展 `DefenseType` → 继承 `DefenseEntity`（仅数据声明） → 写 `[IsAnalyzer(AnalyzerType.Defense)]` 静态方法
4. **新增效果**：写 `[IsAnalyzer(AnalyzerType.DSL)]` 静态方法 → `WriteEffect` 引用 `analyzerKey`
5. **新增动态注册**：准备 `ModifierCallback` 或 `OverrideCallback` → `RegistCallbackOnJudge`
6. **新增攻击阶段钩子**：`WithRuntime(AttackStage.CEValue, analyzerKey)` → 写 `[IsAnalyzer(AnalyzerType.DSL)]` 方法

> **HPB 关键差异**：自定义防御不再需要实现 `Work()` 方法。自定义效果不再传入 `Action<...>` 委托。所有行为通过 `[IsAnalyzer]` 注册为静态方法。

## 13. 排障指南

- **技能按钮不可用**：区分 `Illegal`（未注册）和 `Rejected`（Check 不通过）
- **技能未出现**：最常见——返回了 `DSL.SourceFile` 而非 `IDSLSourceFile`（源生成器报告 Error）
